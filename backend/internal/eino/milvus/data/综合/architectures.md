# 综合：系统架构与工程实践

本专题汇总跨语言、跨组件的系统设计与工程实践，包括架构模式、稳定性、观测性与交付。

## 目录
- 架构模式
- 可观测性
- 稳定性与弹性
- DevOps 与交付
- 安全与合规

## 架构模式
结合业务场景选择分层架构、微服务、事件驱动或混合架构。

### 常见模式
- 分层架构（表现层/领域层/基础设施层）
- 微服务与服务网格（服务拆分、独立部署、网格治理）
- 事件驱动（EDA）、CQRS、事件溯源（Event Sourcing）
- Serverless 与 FaaS（弹性、按需计费）

### 场景选择
- 复杂领域与团队规模大：微服务 + 领域驱动设计（DDD）
- 强一致交易：单体或有界上下文内事务 + Saga/补偿
- 分析/离线：数据湖/仓 + 流批一体（如 Flink）

## 可观测性
- 三大支柱：日志、指标、链路追踪
- 指标：RED（Rate/Error/Duration）、USE（Utilization/Saturation/Errors）
- 工具：OpenTelemetry、Prometheus、Loki、Tempo/Grafana、Jaeger
- 追踪上下文标准：W3C Trace Context；跨边界传递

## 稳定性与弹性
- 模式：重试（含退避抖动）、限流、熔断、舱壁隔离、超时
- 反压：令牌桶/漏桶、背压协议、丢弃策略
- 混沌工程：故障注入、演练日常化；故障预案与演练
- 灰度发布：金丝雀、蓝绿、分流、回滚阈值

## DevOps 与交付
- CI/CD：多分支策略、预提交检查、金丝雀/自动回滚
- IaC：Terraform/Pulumi；声明式配置与漂移检测
- 镜像与供应链安全：SBOM、签名与镜像准入
- 环境一致性：容器化、配置中心、密钥管理

## 安全与合规
- 身份与访问：OAuth2/OIDC、RBAC、零信任
- 数据：静态/传输加密、密钥轮换、最小权限
- 审计与合规：GDPR、PCI-DSS、日志留存/脱敏策略

## 运行手册要点
- SLO/SLA/SLI 设定与预算错误率
- 值班与告警分级、排班与交接
- 事后复盘（Postmortem）与改进闭环
